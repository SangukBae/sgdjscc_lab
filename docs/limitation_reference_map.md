# SGD-JSCC 한계점 우선순위 지도

ETRI 과제 초점은 최대 `PSNR`이 아니라 **무선 전송 후 시맨틱 의도의 신뢰성 있는
보존**이다. 이 기준에서 SGD-JSCC 한계점을 우선순위화한다.

## 우선순위 한계점

| 한계 | 내용 | `sgdjscc_lab` 대응 |
|---|---|---|
| **A** | 시맨틱 가이드 하 할루시네이션·시맨틱 불일치 | 강한 시맨틱 검증, 할루시네이션 탐지, 적응형 가이드, 신뢰할 수 있는 SRS |
| **B** | 시맨틱 부가정보의 취약성·전송 오버헤드 | 손상 캡션/엣지 견고성, 선택적 전송, 패킷 인식 regeneration |
| **C** | 높은 확산 복원 비용·디코딩 지연 | few-step 디코딩, consistency 가속, 실용적 지연 |
| **D** | 페이딩 하 약한 채널 적응·강한 CSI 가정 | 채널 조건화, 블라인드 복원, 채널 토큰 조건화 |
| E | `128×128` 넘어선 좁은 평가 범위 | 시간 인식 파이프라인, 패킷 손상, 더 넓은 시맨틱 평가 |
| F | MIMO/OFDM/다중 사용자 미포함 | 핵심 신뢰도 프레임워크 안정 후 후속 확장 |

`A`가 신뢰도 목표에 가장 직접 반하므로 최우선, `B`(부가정보 견고성) → `C`(지연) →
`D`(페이딩)가 뒤따른다. `E/F`는 핵심 트랙 이후의 확장이다.

## 우선순위 근거

- **A**: 캡션·엣지·확산 가이드가 틀리면 시각적으로 자연스럽지만 시맨틱적으로 틀린
  출력(할루시네이션)을 만든다. 고 SNR에서도 가이드가 충실 복원을 방해할 수 있다.
  프레임워크가 할루시네이션·SRS를 명시 평가하므로 핵심 ETRI 문제다.
- **B**: 캡션·엣지맵 전송 오류는 디코더를 오도하고 조건화를 무효화한다. lab은 이미
  캡션 토큰·엣지 손상을 모델링하므로 일급 신뢰도 이슈다.
- **C**: 다단계 디노이징이 주요 런타임 병목이다. 제한된 SRS 손실로 더 적은 스텝에서
  동작하는 품질-지연 트레이드오프가 필요하다.
- **D**: 현실 무선 평가에는 블라인드/불완전 CSI 견고성이 필요하지만, 핵심
  시맨틱·지연 작업 이후에 온다.

## 낮은 우선순위 (실재하나 첫 차단 요소는 아님)

- `PSNR`이 항상 최고는 아님 — ETRI 목표는 픽셀 완벽 복원이 아니라 시맨틱 신뢰도.
- 학습 비용·데이터셋 규모 — 재현성·배포에 영향은 있으나 A/B/C보다 덜 직접적.

## 워크스트림 매핑

| 워크스트림 | 계획 역할 |
|---|---|
| 패킷 인식 시맨틱 제어·regeneration | 적응형 가이드 강도, 패킷 검증기, 시맨틱 재시도 정책 |
| 저지연 확산 디코딩 | DDIM ablation, few-step, consistency 가속, 지연 프로파일링 |
| 더 강한 시맨틱 검증 | 할루시네이션 점검, SRS-v2, 검증된 신뢰도 기반 최종 출력 선택 |
| CSI-free/블라인드 채널 조건화 | 수신기 evidence 추상화, 블라인드 복원, 채널 토큰 조건화 |
| 더 넓은 채널·시스템 확장 | 페이딩 견고성, 시간 인식 평가, 이후 MIMO/OFDM |
